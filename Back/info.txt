Para instalar todas as dependências é necessário digitar: npm install no powerShell

Se quiser colocar o powerShell como default no terminal do vscode é necessário abrir o command palett: ctrl + shift + p
e colar a seguinte informação: Terminal: Select Default Profile
depois disso, aperte enter e seleciona o powerShell

dependências utilizadas:
  - npm install express --save
  - npm install mysql
  - npm install nodemon -g <------- único necessário instalar, o resto só dar npm install para poder instalar tudo
  - npm install --save sequelize
  - npm install --save mysql2
  - npm install body-parser <------ transforma o body do html em um json, não sei se vai ser necessário utilizar isso mais tarde


O nodemon serve para atualizar o server a qualquer mudança, não é necessário ficar abrindo e fechando o servidor.
Para acessar pelo nodemon é só escrever: nodemon start - informação pode ser modificada no package.json
Como o nodemon executa um script detro do powerShell, é necessário abrir o powerShell como administrador e executar:
Set-ExecutionPolicy Remotesigned
Assim é permitido utilizar scripts assinados digitalmente


=======

Sobre a parte de models, as partes comentadas ".sync" devem ser descomentadas caso as tabelas sumam do MySQL. Elas forçam que sejam criadas ou excluídas. Ou seja, se ela estiver criada e quando rodar o servidor estiver sem comentar, ela irá excluir a tabela e dar erro.

Caso ela não exista, é só descomentar que irá criar novamente.
